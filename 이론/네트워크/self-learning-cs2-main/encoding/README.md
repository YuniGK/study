## Base 64 인코딩

base64 인코딩은 문자(아스키 코드), 이미지 등의 데이터를 다음과 같은 64가지 문자로 구성된 코드로 변환하기 위해 사용됩니다. 

![base64 table](/static/base64.png)

base64 인코딩은 문자, 이미지 등의 데이터를 다음과 같은 64가지 문자로 구성된 코드로 변환하기 위해 사용됩니다. base64는 ‘64진법’을 의미합니다. base64 인코딩을 직역하면 64진법 인코딩입니다. 그래서 하나의 base64 인코딩값을 표현하기 위해 64개의 문자가 사용됩니다. 

64(2^6)진수 하나를 표현하기 위해 6비트가 필요하기에, 문자, 이미지 등의 데이터를 6비트씩 나누어 변환하게 됩니다. 6개의 비트로 나뉜 데이터는 기본적으로 네 개씩(24비트씩) 변환되지요.



예를 들어 'abc'라는 문자열이 있다고 가정해 봅시다. a, b, c는 각각 8비트 아스키 코드 01111000, 01111001, 01111010 표현됩니다. 즉, abc는 총 24비트 코드로 인코딩 될 수 있습니다. 여기서 6비트씩 끊어 위 base64 대응표에 맞게 변환한 결과는 eHl6가 됩니다. 


![base64 example1](/static/base64-ex1.png)

꼭 6비트씩 나누어떨어지지 않는 경우도 있습니다. 예를 들어 'xy'의 경우 총 16비트로 6비트씩 네 개로 나누어떨어지지 않지요.

이 경우 0으로 채워진 패딩(padding)이 채워지고, 이는 =로 인코딩됩니다. 
'xy'는 총 16비트로 표현되고, 6비트 네 개씩 변환되기에는 비트 수가 부족합니다. 따라서 부족한 비트는 0으로 간주되고, =로 인코딩됩니다. 따라서 최종 결과는 다음 그림과 같이 eHk=가 됩니다.


![base64 example2](/static/base64-ex2.png)

base64 인코딩은 다양한 프로그래밍 언어에서 제공되는 기능이기도 하고, 온라인[1]에서도 쉽게 실습이 가능하니, 참고 바랍니다.


[1] https://www.base64encode.org/
